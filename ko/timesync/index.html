TimeSync

PORT TimeSync Client는 공장 네트워크의 모든 PC 시계를 일치시키는 경량 Windows 트레이 애플리케이션입니다. 하나 이상의 SNTP 타임 호스트에 대한 시계 오프셋을 모니터링하고, 직접 타임 호스트로 동작할 수 있으며, mDNS로 LAN의 호스트를 자동으로 찾아냅니다 — 설정 파일도, 런타임 의존성도 필요 없습니다.

다운로드​

이름버전OS요구 사항링크
TimeSync Client (포터블)v1.0.0Windows x64없음 — 단일 exeTimeSyncClient.zip

:::tip 포터블 단일 파일 다운로드는 자체 완결형 실행 파일 하나입니다. 설치 프로그램, .NET 런타임, 추가 DLL이 전혀 필요 없습니다 — 압축을 풀고 TimeSyncClient.exe를 실행하면 됩니다. :::

주요 기능​

  • 시계 오프셋 모니터링 — 등록된 각 호스트를 SNTP(RFC 5905)로 폴링하여 호스트별 Offset (ms), RTT (ms), Stratum을 실시간 표시합니다.
  • 시스템 트레이 — 트레이 아이콘이 최악 상태 기준으로 색이 바뀌고 (녹색 = 정상, 주황 = 임계값 초과, 빨강 = 무응답), 호스트 상태가 나빠지면 풍선 알림을 띄웁니다. 창을 닫으면 트레이로 숨겨집니다.
  • 호스트 자동 검색 (mDNS) — _porttime._udp.local을 광고하는 호스트가 Discovered hosts 목록에 자동으로 나타납니다. 선택 후 Add를 누르면 바로 모니터링이 시작됩니다. LAN의 PORT 타임 호스트와 즉시 연동됩니다.
  • 내장 타임 호스트 — 이 PC를 SNTP 타임 호스트로 사용할 수 있습니다 (기본 바인드 0.0.0.0:123). 구동 중에는 mDNS로 자신을 광고하여 다른 클라이언트가 자동으로 발견합니다.
  • Windows 시작 시 자동 실행 — 사용자별 자동 시작 옵션 (트레이로 최소화 시작).

빠른 시작​

  1. TimeSyncClient.zip을 내려받아 압축을 풉니다.
  2. TimeSyncClient.exe를 실행합니다. 메인 창이 열리고 트레이에도 아이콘이 생깁니다.
  3. 잠시 기다리면 LAN의 타임 호스트가 **Discovered hosts (mDNS)**에 나타납니다. 선택 후 Add selected를 누르거나, 192.168.10.10:123 형식의 주소를 직접 입력해 Add를 누릅니다.
  4. Hosts 그리드에서 호스트별 오프셋, 왕복 지연, Stratum, 마지막 확인 시각을 실시간으로 확인합니다.
  5. 이 PC를 타임 호스트로 쓰려면 Settings에서 Run local SNTP host를 켜고 Save를 누릅니다.

설정​

설정기본값설명
Poll interval30초전체 호스트 자동 폴링 간격
Timeout1000 msSNTP 질의당 수신 타임아웃
Warn offset100 ms호스트를 주황색으로 바꾸고 알림을 띄우는 절대 오프셋
Run local SNTP host꺼짐아래 바인드 주소로 타임 호스트 구동
Host bind0.0.0.0:123내장 타임 호스트의 바인드 주소
Start with Windows꺼짐로그온 시 사용자별 자동 시작 (트레이로 시작)
Start minimized꺼짐창 대신 트레이로 숨겨진 채 시작

설정은 %AppData%\PortTimeSync\settings.json에 저장됩니다.

동작 원리​

  • SNTP — 각 폴링은 48바이트 요청/응답 한 번의 교환입니다. RFC 5905의 4개 타임스탬프로 오프셋과 지연을 계산하고, Originate Timestamp가 요청과 일치하지 않는 응답은 거부합니다(오래된/위조 응답 방지). 오프셋이 양수이면 로컬 시계가 호스트보다 느린 것입니다.
  • mDNS 검색 — 호스트는 서비스 타입 _porttime._udp.local(RFC 6762/6763)을 광고합니다. 클라이언트는 지속적으로 브라우징하여 각 인스턴스를 ip:port로 해석합니다. 다중 NIC 환경을 지원합니다: 모든 IPv4 인터페이스에서 멀티캐스트 그룹에 조인합니다.
  • 방화벽 참고 — 내장 호스트는 UDP 123을 수신하고, mDNS는 UDP 5353(멀티캐스트 224.0.0.251)을 사용합니다. 호스트가 검색되지 않으면 개인/도메인 프로필에서 이 포트들의 인바운드를 허용하세요.

:::note 모니터링 전용 TimeSync Client는 시계 오프셋을 보고만 하며 Windows 시스템 시계를 변경하지 않습니다. 측정된 오프셋을 바탕으로 OS 시간 서비스(예: w32tm)나 운영자 조치로 시계를 보정하세요. :::
