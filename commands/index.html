    PORT Command Reference¶

    This document provides a comprehensive reference for all available PORT commands based on the InputCommand function analysis.

    Script Management Commands¶

    script - Script Execution¶

    Usage: script <script_file> <function_name> [args...]

    Description: Executes JavaScript (.js) or rule (.rule) files with specified functions and arguments.

    Features: - Supports both .js and .rule files - Absolute and relative path resolution - Timeout support with --timeout flag - Async execution with cancellation support - Boot script support for special boot.js files - Context-aware execution with server integration

    Examples: 

    script ./automation.js initialize
script /path/to/boot.js startup --timeout 5000

    cancel - Cancel Script Execution¶

    Usage: cancel <script_file> <function_name>

    Description: Cancels a running script execution by its unique identifier.

    Features: - Cancels async script execution - Uses script file and function name as identifier - Immediate termination of running scripts

    Version & Information Commands¶

    version - Display Version Information¶

    Usage: version

    Description: Displays the current PORT version information.

    Output: Returns formatted version string with build information.

    env - Environment Information¶

    Usage: env

    Description: Displays environment variables and system information.

    Output: - VERSION: Current PORT version - HOMEPATH: Executable directory path

    Project Management Commands¶

    run - Run Project¶

    Usage: run [project_name]

    Description: Starts and runs a PORT project with specified configuration.

    Features: - Auto-detects project name from current directory if not specified - Supports various flags for configuration override: - --address: Override listen address - --port: Override listen port - --call-process: Override call application name - --ng ignore: Allow NG (Not Good) status - --sync <directory>: Set sync directory - --ping-address <host:port>: Enable ping-pong with endpoint - --in-packages <packages>: Comma-separated package list - --new-msg: Enable updated message handling

    Examples: 

    run MyProject --port 8080 --address 0.0.0.0
run --ping-address localhost:9090 --sync ./sync_dir

    new - Initialize New Project¶

    Usage: new <project_name>

    Description: Creates a new PORT project with initial directory structure and configuration files.

    Created Structure: - port/ - Main configuration directory - port/app/ - Application files - port/app/ecma5/ - ECMAScript 5 files - port/app/mqtt/ - MQTT configuration - port/app/gem/ - GEM (Generic Equipment Model) files - boot.js - Boot script - .enum - Enumeration definitions - .rule - Rule definitions - Various GEM configuration files (.gem, .sv, .dvv, .al, .ce, .rid, .ecv)

    push - Push Project to Repository¶

    Usage: push [project_name]

    Description: Pushes current project changes to the repository with change detection.

    Features: - Automatic change detection for multiple file types - Changelog generation in .port.md file - Supports various file extensions: .logger, .cctv, .pub, .js, .rule, .al, .ce, .dvv, .ecv, .sv, .gem, .msg - Message collector generation - Automatic database updates

    pull - Pull Project from Repository¶

    Usage: pull [project_name]

    Description: Pulls project files from repository to local directory.

    Features: - Pulls all project file types from repository - Updates local directory structure - Synchronizes configuration files - Updates .NET project files

    Listing Commands¶

    ls - List Resources¶

    Usage: ls <resource_type> [additional_args]

    Description: Lists various system resources based on the specified type.

    Resource Types:

    ls user - List Users¶

    Lists all registered usernames in the system.

    ls com - List COM Ports¶

    Lists all available serial communication ports.

    ls tcp [address] - List TCP Addresses¶

    Lists TCP addresses for the specified host (default: 0.0.0.0). Performs DNS lookup to resolve host addresses.

    ls repo - List Repositories¶

    Lists all available repositories with timestamps. Output Format: repository_name modification_time

    ls pkg - List Packages¶

    Lists all available packages (.pkg files) with timestamps. Output Format: package_name modification_time

    Package Management Commands¶

    save - Save Package¶

    Usage: save <package_name>

    Description: Creates a package (.pkg) file from DLL files in the current directory.

    Features: - Scans current directory for .dll files - Compresses files into .pkg format - Supports --path flag to specify source directory - Automatic package directory creation

    Examples: 

    save MyPackage --path /path/to/dlls

    pack / build - Build Package¶

    Usage: pack <dll_file> <output_name>

    Description: Builds a package from a DLL file with full initialization and testing.

    Features: - Loads and initializes DLL package - Extracts message definitions (GET, SET, MAPPING) - Sets memory data allocation - Runs initialization with arguments - Generates comprehensive build report - Supports --args flag for initialization arguments

    Examples: 

    pack MyLibrary.dll MyPackage --args "config=production"

    clear - Clear Packages¶

    Usage: clear

    Description: Clears all loaded packages from memory and terminates active processes.

    User Management Commands¶

    login - User Login¶

    Usage: login <username> <password>

    Description: Authenticates user and enters interactive command mode.

    Features: - User authentication - Interactive command prompt - Session-based command execution

    add - Add Resources¶

    Usage: add [flags]

    Description: Adds various resources to the system based on specified flags.

    Flags:

    --user <username> <password> - Add User¶

    Adds a new user to the system with specified credentials.

    --enum <enum_definitions> - Add Enumerations¶

    Adds enumeration definitions to the .enum file. Format: type name:value name:value (comma-separated for multiple) Example: --enum "Status RUNNING:1 STOPPED:0,Mode AUTO:1 MANUAL:0"

    --group <group_names> [message_file] - Add Groups¶

    Creates new groups with optional message file specification. Features: - Creates group directories - Generates .msg files for each group - Supports comma-separated group names

    Repository Management Commands¶

    repo - Repository Operations¶

    Usage: repo <file_path>

    Description: Manages repository operations with upload capabilities.

    Flags:

    --upload local - Local Upload¶

    Uploads files from local directory to package repository. Process: - Copies files from source directory - Creates target directory in package repository - Preserves file structure and permissions

    --upload remote - Remote Upload¶

    Handles remote repository upload operations.

    Server Communication Commands¶

    latency - Measure Latency¶

    Usage: latency

    Description: Measures network latency to the server using timestamp-based calculation.

    flow - Flow Status¶

    Usage: flow

    Description: Retrieves current flow status from the server.

    uptime - Server Uptime¶

    Usage: uptime

    Description: Gets server uptime information.

    date - Server Date¶

    Usage: date

    Description: Retrieves current server date and time.

    status - Server Status¶

    Usage: status

    Description: Gets comprehensive server status information.

    queue - Queue Management¶

    Usage: queue <operation>

    Description: Manages server message queues.

    stack - Stack Management¶

    Usage: stack <operation>

    Description: Manages server stack operations.

    storage - Storage Management¶

    Usage: storage <operation>

    Description: Manages server storage operations.

    list - List Server Resources¶

    Usage: list <resource_type>

    Description: Lists various server resources.

    get - Get Server Data¶

    Usage: get <resource_identifier>

    Description: Retrieves specific data from the server.

    set - Set Server Data¶

    Usage: set <group> <key> <value>

    Description: Sets data on the server with group, key, and value specification.

    shutdown - Server Shutdown¶

    Usage: shutdown <confirmation>

    Description: Initiates server shutdown sequence.

    request-app - Request Application¶

    Usage: request-app [parameters]

    Description: Sends application request to the server.

    Utility Commands¶

    sleep - Sleep/Wait¶

    Usage: sleep

    Description: Pauses execution (utility command for scripting).

    help - Display Help¶

    Usage: help

    Description: Displays comprehensive help information for all available commands.

    Command Flags Reference¶

    Global Flags¶

    • --timeout <milliseconds>: Set execution timeout
    • --address <address>: Override listen address
    • --port <port>: Override listen port
    • --path <path>: Specify file path
    • --args <arguments>: Pass initialization arguments
    • --ng <ignore>: Allow NG status
    • --sync <directory>: Set sync directory
    • --ping-address <host:port>: Enable ping-pong
    • --in-packages <packages>: Specify package list
    • --new-msg: Enable updated message handling
    • --upload <local|remote>: Upload mode
    • --user <username> <password>: User operations
    • --enum <definitions>: Enumeration definitions
    • --group <names>: Group operations

    Error Handling¶

    The system provides comprehensive error handling for: - Invalid command syntax - Missing required arguments - File system errors - Network communication errors - Authentication failures - Resource not found errors - Permission errors

    Examples¶

    Basic Project Workflow¶

    # Create new project
new MyProject

# Initialize and run
run MyProject --port 8080

# Make changes and push
push MyProject

# Pull updates
pull MyProject

    Package Development¶

    # Build package from DLL
pack MyLibrary.dll MyPackage --args "config=test"

# Save package
save MyPackage --path ./output

# Clear packages
clear

    System Administration¶

    # Add user
add --user admin password123

# Check server status
status

# Measure latency
latency

# List repositories
ls repo
