# TimeSync

**PORT TimeSync Client** は、工場ネットワーク上のすべての PC の時計を一致させる
軽量な Windows トレイアプリケーションです。1 つ以上の SNTP タイムホストに対する
時計オフセットを監視し、自身がタイムホストとして動作でき、mDNS により LAN 上の
ホストを自動検出します — 設定ファイルもランタイム依存も不要です。

## ダウンロード

| 名前 | バージョン | OS | 要件 | リンク |
|------|-----------|----|------|--------|
| TimeSync Client (ポータブル) | v1.0.0 | Windows x64 | なし — 単一 exe | [TimeSyncClient.zip](pathname:///file/TimeSyncClient.zip) |

:::tip ポータブル単一ファイル
ダウンロードは自己完結型の実行ファイル 1 つです。インストーラーも .NET ランタイムも
追加 DLL も不要 — 解凍して `TimeSyncClient.exe` を実行するだけです。
:::

## 主な機能

- **時計オフセット監視** — 登録した各ホストを SNTP (RFC 5905) でポーリングし、
  ホストごとの *Offset (ms)*、*RTT (ms)*、*Stratum* をリアルタイム表示します。
- **システムトレイ** — トレイアイコンは最悪のホスト状態で色が変わり
  (緑 = 正常、橙 = しきい値超過、赤 = 無応答)、状態悪化時にバルーン通知を表示します。
  ウィンドウを閉じるとトレイに格納されます。
- **ホスト自動検出 (mDNS)** — `_porttime._udp.local` を広告するホストが
  *Discovered hosts* リストに自動表示されます。選択して **Add** を押すだけで
  監視を開始できます。
- **内蔵タイムホスト** — この PC を SNTP タイムホストとして動作させられます
  (既定バインド `0.0.0.0:123`)。動作中は mDNS で自身を広告します。
- **Windows 起動時に自動実行** — ユーザーごとの自動起動オプション (トレイ格納で開始)。

## クイックスタート

1. [TimeSyncClient.zip](pathname:///file/TimeSyncClient.zip) をダウンロードして解凍します。
2. `TimeSyncClient.exe` を実行します。メインウィンドウが開き、トレイにも表示されます。
3. しばらく待つと LAN 上のタイムホストが **Discovered hosts (mDNS)** に表示されます。
   選択して **Add selected** を押すか、`192.168.10.10:123` 形式のアドレスを入力して
   **Add** を押します。
4. **Hosts** グリッドでホストごとのオフセット、RTT、Stratum、最終確認時刻を確認します。
5. この PC をタイムホストにするには、*Settings* で **Run local SNTP host** を有効にして
   **Save** を押します。

## 設定

| 設定 | 既定値 | 説明 |
|------|--------|------|
| Poll interval | 30 秒 | 全ホストの自動ポーリング間隔 |
| Timeout | 1000 ms | SNTP クエリごとの受信タイムアウト |
| Warn offset | 100 ms | ホストを橙色にし通知を出す絶対オフセット |
| Run local SNTP host | オフ | 下記バインドアドレスでタイムホストを起動 |
| Host bind | `0.0.0.0:123` | 内蔵タイムホストのバインドアドレス |
| Start with Windows | オフ | ログオン時のユーザーごと自動起動 (トレイで開始) |
| Start minimized | オフ | ウィンドウを表示せずトレイに格納して開始 |

設定は `%AppData%\PortTimeSync\settings.json` に保存されます。

## トレイメニュー

| 項目 | 動作 |
|------|------|
| **Open** (ダブルクリック) | メインウィンドウを復元 |
| **Sync now** | 全ホストを即時ポーリング |
| **Run local host** | 内蔵 SNTP ホストの切り替え |
| **Exit** | ホストを停止して終了 |

## 動作の仕組み

- **SNTP** — 各ポーリングは 48 バイトの要求/応答 1 往復です。RFC 5905 の 4 つの
  タイムスタンプからオフセットと遅延を計算し、*Originate Timestamp* が要求と一致
  しない応答は拒否します。オフセットが正ならローカル時計がホストより遅れています。
- **mDNS 検出** — ホストはサービスタイプ `_porttime._udp.local` (RFC 6762/6763) を
  広告します。マルチホーム環境に対応し、すべての IPv4 インターフェイスで
  マルチキャストグループに参加します。
- **ファイアウォール** — 内蔵ホストは UDP 123 を待ち受け、mDNS は UDP 5353
  (マルチキャスト `224.0.0.251`) を使用します。ホストが検出されない場合は
  *プライベート/ドメイン* プロファイルでこれらの受信を許可してください。

:::note 監視専用
TimeSync Client は時計オフセットを報告するのみで、Windows システム時計は変更
しません。測定されたオフセットに基づき、OS の時刻サービス (例: `w32tm`) や
オペレーター操作で時計を補正してください。
:::
