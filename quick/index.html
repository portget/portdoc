    Quick Start¶

    A step-by-step guide to building a .NET equipment application with Port.

    Requirements¶

    Requirement Specification
    OS Windows 11+ / Windows Server 2003+
    Memory Minimum 32GB
    .NET SDK .NET 8.0+
    Port v1.1.78 Download
    NuGet portdic package

    Step 1. Create a New Project¶

    dotnet new wpf -n MyEquipment
cd MyEquipment
dotnet add package portdic

    dotnet new winforms -n MyEquipment
cd MyEquipment
dotnet add package portdic

    dotnet new console -n MyEquipment
cd MyEquipment
dotnet add package portdic

    Step 2. Create Repository¶

    Create a repository with Port CLI¶

    port new myequipment

    This creates the project structure for storing .page files.

    Writing .page files¶

    .page files define equipment data structures. Each line follows the format: Key DataType pkg:PackageName property:{JSON}.

    LP1_Cont_o              enum.OffOn  pkg:LoadPort
LP1_Configure_Value_i   num         pkg:LoadPort
LP1_Main_Air_i          num         pkg:LoadPort
IO_WTR_Upper_Arm_Vac_Detect_1_1_i  enum.OffOn  pkg:WTR
IO_WTR_Lower_Arm_Vac_Detect_1_1_i  enum.OffOn  pkg:WTR

    Auto-generate from Documents¶

    You can auto-generate .page files and C# constant classes from Word/Excel documents.

    // Model: maps document columns to entry fields
public class IOModel
{
    [ColumnHeader, EntryProperty]
    public string Pin_No { set; get; } = string.Empty;

    [ColumnHeader, EntryProperty]
    public string Port_NO { set; get; } = string.Empty;

    [ColumnHeader, EntryKey]
    public string Description { set; get; } = string.Empty;

    [ColumnHeader, EntryProperty]
    public string Model { set; get; } = string.Empty;

    [ColumnHeader, EntryProperty]
    public string Bit_On { set; get; } = string.Empty;

    [EntryDataType]
    public string DataType { set; get; } = "enum.OffOn";
}

// Converter: reads docx, outputs .page + .cs
[Document(@"D:\project\IO_Map.docx")]
public class IODocument
{
    [Save(@"D:\project\port\device\io.page", @"D:\project\port\io1.cs")]
    public Document<IOModel> Convert(Document<IOModel> doc)
    {
        doc.ForEach(v => v.DataType = "enum.OffOn");
        doc.ForEach(v => v.Package = "DigitalIO.DI");
        return doc;
    }
}

    Generated io.page: 

    Main_CDA_Pressure_Switch_Check  enum.OffOn  pkg:DigitalIO.DI  property:{"Pin_No":"1","Port_NO":"X01.00","Model":"ISE40A-C6-R-F","Bit_On":"Sensing"}
Door_Lock_On_Check              enum.OffOn  pkg:DigitalIO.DI  property:{"Pin_No":"6","Port_NO":"X01.05","Model":"G7SA-2A2B","Bit_On":"Lock"}

    Generated io1.cs: 

    // Auto-generated by Port. Do not edit manually.
public static class EFEM
{
    public const string Main_CDA_Pressure_Switch_Check = "EFEM.Main_CDA_Pressure_Switch_Check";

}

    The generated constant class provides type-safe keys for [Binding] attributes and Port.Set()/Port.Get() calls.

    See Document Attributes for the complete attribute reference.

    Step 3. Push to Repository¶

    port push

    Parses .page files and registers them in Port's in-memory database.

    Step 4. Initialize Port¶

    Initialization order¶

    Port.Repository.Load() → Port.Add<T>() → Port.OnReady += → Port.Run()

    WPF Application¶

    using portdic;
using System.Windows;

namespace MyEquipment
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent();

            if (Port.Repository.Load("myequipment"))
            {
                // 1. Register packages
                Port.Add<GemHelper>("GEM");
                Port.Add<Bulb>("DigitalIO");

                // 2. Register FlowControllers with Models
                Port.Add<LoadportController, LoadportModel>("LP1");
                Port.Add<LoadportController, LoadportModel>("LP2");
                Port.Add<WTRController, WTRModel>("WTR");

                // 3. Register TransferController
                Port.Add<TransferController>("Transfer");

                // 4. Subscribe to ready event and start
                Port.OnReady += Port_OnReady;
                Port.Run();
            }
        }

        private void Port_OnReady(object? sender, EventArgs e)
        {
            // Port server synchronized — Set/Get operations are now available
        }
    }
}

    Initialization Order

    You must follow the order: Port.Repository.Load() → Port.Add<T>() → Port.OnReady += → Port.Run(). Once Port.Run() is called, the server starts. The OnReady event fires when synchronization is complete.

    Port.Add overloads¶

    Call Purpose
    Port.Add<T>(key) Register a Package, GEM, or TransferController
    Port.Add<TController, TModel>(key) Register a FlowController + Model pair

    Port.Add<TController, TModel>("LP1") internally:

    1. Creates a LoadportController instance with the key "LP1"
    2. Activates the [Binding] entries matching "LP1" in the LoadportModel
    3. Auto-registers nested [Flow] classes (e.g., FoupLoadFlow)

    You can register the same Controller/Model type multiple times with different keys (e.g., LP1, LP2).

    Step 5. Create a Package¶

    Use the [Package] attribute to define equipment libraries. Port automatically manages instances and generates APIs.

    using portdic;

[Package]
public class IODevice
{
    [EntryProperty]
    public IProperty property { get; set; } = null!;

    [API]
    public double DI { set; get; }

    [API]
    public double Digital { set; get; }

    [API]
    public double Analog { set; get; }
}
    Attribute Purpose
    [Package] Registers the class as a Port-managed package
    [EntryProperty] Injects IProperty for entry property access
    [API] Generates a REST API endpoint automatically
    [Logger] Injects ILogger automatically

    See the full attribute list in the Attribute Reference.

    Step 6. Create FlowController + Model¶

    A FlowController defines equipment automation flows. A Model manages data bindings per controller instance.

    Architecture¶

    ┌─ LoadportController ──────────────────────┐
│                                           │
│  ┌─ FoupLoadFlow ──────────────────────┐  │
│  │  [FlowStep] StatusCheck(model)      │  │
│  │  [FlowStep] SendLoadCommand(model)  │  │
│  │  [FlowStep] FoupLoadCompleted(model)│  │
│  └─────────────────────────────────────┘  │
│                                           │
│  ┌─ FoupUnloadFlow ────────────────────┐  │
│  │  [FlowStep] StatusCheck()           │  │
│  │  [FlowStep] SendUnloadCommand()     │  │
│  │  [FlowStep] FoupUnloadCompleted()   │  │
│  └─────────────────────────────────────┘  │
└───────────────────────────────────────────┘
         ↕ [Binding]
┌─ LoadportModel ───────────────────────────┐
│  LP_Cont_o            → LP1 or LP2 key    │
│  LP_Configure_Value_i → LP1 or LP2 key    │
│  LP_Main_Air_i        → LP1 or LP2 key    │
└───────────────────────────────────────────┘

    Define a Model¶

    A [Model] class uses [Binding] attributes to map data per controller instance.

    using portdic;

[Model]
public class LoadportModel
{
    [Binding("LP1", EFEM.LP1_Cont_o)]
    [Binding("LP2", EFEM.LP2_Cont_o)]
    public Entry LP_Cont_o { set; get; }

    [Binding("LP1", EFEM.LP1_Configure_Value_i)]
    [Binding("LP2", EFEM.LP2_Configure_Value_i)]
    public Entry LP_Configure_Value_i { set; get; }

    [Binding("LP1", EFEM.LP1_Main_Air_i)]
    [Binding("LP2", EFEM.LP2_Main_Air_i)]
    public Entry LP_Main_Air_i { set; get; }
}

    When multiple [Binding] attributes are declared on a single Model class, calling Port.Add<LoadportController, LoadportModel>("LP1") activates only the bindings matching "LP1".

    Define a FlowController¶

    using portdic;

[FlowController]
public class LoadportController
{
    [Flow("Load")]
    public class FoupLoadFlow
    {
        [FlowHandler]
        public IFlowHandler Handler { set; get; } = null!;

        [Preset]
        public void Preset()
        {
            // Flow initialization logic
        }

        [FlowStep]
        [FlowWatcherCompare("LP1", EFEM.LP1_Main_Air_i, ">=", 0)]
        [FlowWatcherCompare("LP2", EFEM.LP2_Main_Air_i, ">=", 0)]
        [Timeout(1000, -1)]
        public void StatusCheck(LoadportModel m)
        {
            // Access bound Entry values through the Model
            var value = m.LP_Configure_Value_i.Value.String();
            Handler.Next();
        }

        [FlowStep]
        public void SendLoadCommand(LoadportModel m)
        {
            var air = m.LP_Main_Air_i.Value.String();
            Handler.Next();
        }

        [FlowStep]
        public void FoupLoadCompleted(LoadportModel m)
        {
            var status = m.LP_Cont_o.Value.String();
            Handler.Next();
        }
    }

    [Flow("Unload")]
    public class FoupUnloadFlow
    {
        [FlowStep]
        public void StatusCheck() { }

        [FlowStep]
        public void SendUnloadCommand() { }

        [FlowStep]
        public void FoupUnloadCompleted() { }
    }
}
    Attribute Purpose
    [FlowController] Registers the class as a Flow controller
    [Flow("name")] Registers a nested class as a named Flow
    [FlowHandler] Injects IFlowHandler automatically
    [FlowStep] Defines a Flow execution step (runs in declaration order)
    [FlowWatcherCompare(ctrl, key, op, val)] Watches a condition — enters step when satisfied
    [Timeout(ms, retryCount)] Step timeout (-1 = infinite retry)
    [Preset] Called during Flow initialization
    [Model] Marks a data binding model class
    [Binding(ctrlKey, entryKey)] Binds an entry per controller instance

    Execute a Flow¶

    // Start a flow
Port.Set("LP1.Load", FlowAction.Executing);
Port.Set("LP2.Load", FlowAction.Executing);

// Abort a flow
Port.Set("LP1.Load", FlowAction.Aborting);

    Step 7. Substrate Transfer¶

    [TransferController] manages wafer/substrate transport. It performs multi-stage process scheduling based on routes.

    using portdic;

[TransferController]
public class TransferController
{
    [TransferHandler]
    public ITransferHandler handler { get; set; } = null!;

    [Preset]
    public void Preset()
    {
        handler.OnSubstrateCompleted += OnSubstrateCompleted;
        handler.OnRequestTransfer += OnRequestTransfer;
        handler.OnFlowFinished += OnFlowFinished;
    }

    // Fired when a substrate completes processing
    private void OnSubstrateCompleted(
        Substrate substrate, SubstrateStatus previous, SubstrateStatus current) { }

    // Executes WTR flow when a transfer is requested
    private void OnRequestTransfer(TransferArgs args)
    {
        Port.Set(EFEM.WTR_MANUALMOVING_TARGET_MODULE, args.Substrate.Route[0].Location);
        Port.Set("WTR.Move", FlowAction.Executing);
    }

    // Advances to the next route after flow completion
    private void OnFlowFinished(TransferArgs args)
    {
        if (args.Key == "WTR.Move")
        {
            args.Substrate.NextRoute();
            handler.RequestTransfer(handler.GetSubstrate());
        }
    }

    // Checks arm availability — determines if transfer is possible
    [Arm("Upper")]
    public int Upper()
    {
        var substrate = handler.GetSubstrate();
        return substrate.Direction switch
        {
            Direction.In  => Port.Get(EFEM.IO_WTR_Upper_Arm_Detect_Status_i) == "Off" ? 1 : -1,
            Direction.Out => Port.Get(EFEM.IO_WTR_Upper_Arm_Detect_Status_i) == "On"  ? 1 : -1,
            _ => -1
        };
    }

    [Arm("Lower")]
    public int Lower()
    {
        var substrate = handler.GetSubstrate();
        return substrate.Direction switch
        {
            Direction.In  => Port.Get(EFEM.IO_WTR_Lower_Arm_Detect_Status_i) == "Off" ? 1 : -1,
            Direction.Out => Port.Get(EFEM.IO_WTR_Lower_Arm_Detect_Status_i) == "On"  ? 1 : -1,
            _ => -1
        };
    }

    // Checks location availability
    [Location("LP1")]
    public int LP1() => -1;

    [Location("Stage1")]
    public int Stage1() => 1;
}

    Register and execute substrates¶

    // Register substrates — transported in route order
handler.ClearSubstrate();
handler.AddSubstrate(
    new Substrate
    {
        LotID = "LOT001",
        Key = "1.1",
        Route = new List<Route>
        {
            new MultiSlotRoute("LP1", 1),
            new ProcessRoute("Stage1", "Recipe1.rec"),
            new ProcessRoute("Stage2", "Recipe2.rec"),
            new MultiSlotRoute("LP1", 1),
        }
    });
handler.Execute("LOT001");
    Attribute Purpose
    [TransferController] Registers a transfer controller
    [TransferHandler] Injects ITransferHandler automatically
    [Arm("name")] Checks robot arm status (1=available, -1=unavailable)
    [Location("name")] Checks station availability (1=available, -1=unavailable)
    Route Type Purpose
    MultiSlotRoute(location, slot) Slot-based location (LoadPort, etc.)
    ProcessRoute(location, recipe) Process location (Stage + Recipe)

    Step 8. SECS/GEM Communication¶

    Use the [GEM] attribute to set up semiconductor equipment communication (SECS-II/HSMS).

    using portdic;
using portdic.secs;

[GEM]
public class GemHelper
{
    [GemHandler]
    public IGemHandler handler { get; set; } = null!;

    [Preset]
    private void Preset()
    {
        handler.SetMode(Mode.Passive);
        handler.SetAddress("127.0.0.1:6000");
        handler.SetDevice(0);
        handler.SetT3(45);

        handler.OnEvent += OnEvent;
        handler.OnStatusChanged += OnStatusChanged;
        handler.OnS10F1_TerminalDisplaySingle += OnTerminalDisplay;
    }

    private void OnEvent(DateTime time, string text)
    {
        // Handle GEM events
    }

    private void OnStatusChanged(DateTime time, string text)
    {
        // Handle communication status changes
    }

    private void OnTerminalDisplay(IGemReplier replier, ReceiveSecsMessage message)
    {
        // Handle terminal message from host
        message.Shared("S10F1");  // Broadcast to other controllers
    }
}

    using portdic;
using portdic.secs;

[GEM]
public class GemController
{
    [GemHandler]
    public IGemHandler handler { set; get; } = null!;

    [Preset]
    public void Preset()
    {
        handler.SetMode(Mode.Active);
        handler.SetAddress("127.0.0.1:6000");
        handler.SetDevice(0);

        handler.OnS1F1_AreYouOnline += OnAreYouOnline;
        handler.OnS10F1_TerminalDisplaySingle += OnTerminalDisplay;
    }

    private void OnAreYouOnline(IGemReplier replier, ReceiveSecsMessage message)
    {
        // Handle S1F1 response
    }

    private void OnTerminalDisplay(IGemReplier replier, ReceiveSecsMessage message)
    {
        message.Shared("S10F1");
    }
}

    Shared Messages¶

    message.Shared("S10F1") broadcasts the message to all controllers that declare [Shared] with the same key.

    [FlowController]
public class LoadportController
{
    [Shared("S10F1")]
    public void OnShared(IShared message)
    {
        var msg = message as ReceiveSecsMessage;
        // Receive GEM messages in a FlowController
    }
}

    Step 9. Set / Get Values¶

    After the OnReady event fires, you can read and write data.

    Set — Write values¶

    // Dot notation
Port.Set("EFEM.LP1_Cont_o", "On");

// Category, key, value
Port.Set("EFEM", "LP1_Cont_o", "On");

// Numeric values
Port.Set("EFEM", "LP1_Configure_Value_i", 42.5);

// Execute a flow
Port.Set("LP1.Load", FlowAction.Executing);

    Get — Read values¶

    // Dot notation
var value = Port.Get("EFEM.LP1_Cont_o");
string status = value.AsString;    // "On"

// Category, key
var temp = Port.Get("EFEM", "LP1_Configure_Value_i");
string text = temp.AsString;       // "42.5"

    Using generated constants¶

    The entry.cs constant class generated by port push provides type-safe key references.

    // Auto-generated entry.cs
public static class EFEM
{
    public const string LP1_Cont_o = "EFEM.LP1_Cont_o";
    public const string LP1_Configure_Value_i = "EFEM.LP1_Configure_Value_i";
    public const string IO_WTR_Upper_Arm_Detect_Status_i = "EFEM.IO_WTR_Upper_Arm_Detect_Status_i";
    // ...
}

    // Usage
Port.Set(EFEM.LP1_Cont_o, "On");
var value = Port.Get(EFEM.LP1_Configure_Value_i).AsString;

    Query entry keys¶

    // Get all keys in a category
var keys = Port.GetEntryKeysWithCategory("EFEM");

// Use for UI polling
foreach (var key in keys)
{
    var val = Port.Get(key);
}

    Step 10. Run¶

    Run via CLI¶

    port run myequipment

    Run as GUI Application¶

    dotnet run --project MyEquipment

    The Port server starts automatically when Port.Run() is called.

    Error inspection¶

    port run myequipment --ng ignore

    Detailed error analysis: http://localhost:5001/api/app/ng/?view=table

    {
        Handler.OnFlowFinished += OnFlowFinished;
    }

    private void OnFlowFinished(object sender, FlowFinishedArgs e)
    {
        // Receives all flow completion events
    }
}
